2. Data NOT Collected

  • Meeting recordings (audio, video, transcripts)
  • Meeting participant lists or attendance data
  • Chat messages within Zoom meetings
  • Screen sharing content
  • Zoom account billing or subscription information

3. Data Access Controls

  • Row Level Security (RLS): Zoom tokens are restricted to the owning user — no other user, including administrators, can read another user's tokens via the API.
  • Server-side only: All Zoom API calls are made from Supabase Edge Functions (Deno runtime). Client-side JavaScript never has access to Zoom tokens.
  • Dual authentication: Meeting creation requires both a valid Supabase JWT session AND a valid per-user Zoom OAuth token.

4. Data Deletion & User Rights

  • Users can disconnect their Zoom account at any time via the IDE settings, which immediately deletes all stored tokens.
  • Expired tokens that fail refresh are automatically purged from the database.
  • Users may request complete data deletion by contacting privacy@managekube.com.

6. Vulnerability Management

  • Dependencies audited regularly via automated tooling.
  • Edge Functions use pinned dependency versions to prevent supply-chain attacks.
  • Security vulnerabilities reported to security@managekube.com are triaged within 24 hours.

7. Compliance

ManageKube's data handling practices align with SOC 2 Type II principles (via Supabase infrastructure), GDPR data minimization requirements, and CCPA user rights provisions.
